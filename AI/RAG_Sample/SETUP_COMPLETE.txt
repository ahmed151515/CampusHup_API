✨ FEATURES
═══════════════════════════════════════════════════════════════════════════

✓ Document Parsing
  - Extracts text from PDF files
  - Reads Excel spreadsheets with structure preservation
  - Intelligent text chunking with overlaps

✓ Embeddings
  - Uses all-minilm (lightweight, 45MB)
  - Fast generation of 384-dimensional vectors
  - Cached embeddings for reuse

✓ Vector Search
  - Faiss index for O(log n) similarity search
  - Persistent storage on disk
  - Configurable top-K retrieval

✓ Response Generation
  - Uses llama3.2 (2GB) for quality answers
  - Contextual answers based on retrieved documents
  - Source attribution for transparency

✓ Multiple Interfaces
  - Interactive CLI prompts
  - REST API with FastAPI
  - Streaming responses for real-time output
  - Swagger UI documentation

═══════════════════════════════════════════════════════════════════════════
⚙️ CONFIGURATION
═══════════════════════════════════════════════════════════════════════════

Edit config.py to customize:

  EMBEDDING_MODEL = "all-minilm:latest"  # Model for embeddings
  LLM_MODEL = "llama3.2:latest"          # Model for answers
  
  CHUNK_SIZE = 512                       # Tokens per chunk
  CHUNK_OVERLAP = 50                     # Token overlap
  
  TOP_K_RESULTS = 5                      # Documents to retrieve
  SIMILARITY_THRESHOLD = 0.5             # Min similarity score
  
  API_PORT = 8000                        # API port

❓ TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════

"Ollama not available"
  → Run: ollama serve (in another window)

"Index not found"  
  → Run: python main.py build-index --data-dir .

"Connection refused"
  → Check Ollama is running: ollama serve

"Slow embedding"
  → Normal on first run. Results are cached for speed.

"FAISS import error"
  → Run: pip install faiss-cpu

💡 TIPS
═══════════════════════════════════════════════════════════════════════════

• First query takes longer (embedding generation + LLM inference)
• Subsequent queries are faster (uses cached embeddings)
• Adjust TOP_K_RESULTS to balance accuracy vs. speed
• Use smaller CHUNK_SIZE for more granular documents
• Use API for programmatic access from other applications
• Check API docs at http://localhost:8000/docs (interactive)
