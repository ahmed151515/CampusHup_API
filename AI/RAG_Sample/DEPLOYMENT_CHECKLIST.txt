Before You Start:
─────────────────────────────────────────────────────────────────────────────
☐ Python 3.8+ installed (python --version)
☐ Ollama installed (ollama --version)
☐ ~2GB free disk space for models
☐ PDF and Excel files in project folder


Installation Phase:
─────────────────────────────────────────────────────────────────────────────
☐ Navigate to: C:\Users\zedan\OneDrive\Documents\RAG_Sample
☐ Install dependencies: pip install -r requirements.txt
  - Wait for completion (should be < 5 minutes)
  - Check for errors - if any, see Troubleshooting section

☐ Verify Python packages installed:
  - Open Python: python
  - Import check: import faiss; import langchain; exit()
  - Should complete without errors


Ollama Setup:
─────────────────────────────────────────────────────────────────────────────
☐ Open NEW PowerShell window
☐ Run: ollama serve
  - Should see: "Listening on 127.0.0.1:11434"
  - Keep this window OPEN (don't close it)

☐ In ANOTHER window, verify models:
  - Run: ollama list
  - Should show:
    • all-minilm:latest (45 MB)
    • llama3.2:latest (2.0 GB)

☐ If models missing, pull them:
  - Run: ollama pull all-minilm
  - Run: ollama pull llama3.2
  - Wait for downloads to complete


System Verification:
─────────────────────────────────────────────────────────────────────────────
☐ Run diagnostic test:

  - Should show:

Troubleshooting Checklist:
─────────────────────────────────────────────────────────────────────────────
Issue: "Ollama not available"
☐ Check: ollama serve running?
☐ Solution: Start ollama serve in new window

Issue: "Index not found"
☐ Check: python main.py check-index
☐ Solution: Run python main.py build-index --data-dir .

Issue: "Module not found"
☐ Check: pip list | grep -E "langchain|faiss|ollama"
☐ Solution: Run pip install -r requirements.txt

Issue: "Slow responses"
☐ Normal on first query (embedding generation)
☐ Subsequent queries should be instant

Issue: "Connection refused"
☐ Check: Ollama is running?
☐ Check: Port 11434 not blocked?
☐ Solution: Restart ollama serve

Issue: "Port already in use"
☐ For API, edit config.py: API_PORT = 8001
☐ Restart API


Performance Optimization:
─────────────────────────────────────────────────────────────────────────────
For faster embedding:
☐ Reduce CHUNK_SIZE in config.py
☐ Reduce number of documents

For better answers:
☐ Increase TOP_K_RESULTS in config.py
☐ Use larger CHUNK_SIZE

For faster API response:
☐ Reduce TOP_K_RESULTS
☐ Decrease SIMILARITY_THRESHOLD
